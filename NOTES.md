# NOTES — MNIST 4-Layer NN Implementation Notes

## Q14 Quantization Scheme — Key Difference from 2-Layer Version

The original 2-layer `neural_net/` code uses a **double-dequantization** scheme:
```c
// ORIGINAL 2-layer (DON'T use this for 4+ layers)
acc += DEQUANTIZE_I(inputs[i] * weights[i]);  // Q14×Q14 / 2^14 = Q14
*output = acc >> BITS;                         // dequantize again → integer
```

This works for 2 layers because the signal only gets shifted twice. But for 4 layers, each `>> 14` dequantization halves the dynamic range, and values quickly collapse to zero.

### New Scheme (4-Layer)

All intermediate values **stay in Q14** throughout:
```c
// NEW 4-layer scheme
long long acc = (long long)bias;                          // Q14 bias
acc += ((long long)inputs[i] * (long long)weights[i]) >> BITS;  // Q14×Q14 >> 14 = Q14
*output = (int)acc;                                       // stays Q14!
```

**No `>> BITS` at the output.** Values flow Q14 → Q14 → Q14 → Q14 across all 4 layers.
The final argmax works directly on Q14 values (scale doesn't affect comparison).

## SystemVerilog Implementation Notes

### Data Path

```
                    Q14         Q14         Q14         Q14
  x_test  ──→ [Layer 0] ──→ [Layer 1] ──→ [Layer 2] ──→ [Layer 3] ──→ argmax
  (784)      ReLU (128)   ReLU (64)    ReLU (32)                    (10→class)
```

### Neuron MAC Unit

```
For each output neuron j:
    acc = bias[j]                           // signed 32-bit Q14
    for i = 0 to input_size-1:
        acc += (input[i] * weight[j][i]) >>> 14  // arithmetic right shift
    output[j] = acc                         // Q14
```

**Critical: use a 64-bit accumulator** for the multiply-accumulate:
- `input[i]` is 32-bit signed (Q14)
- `weight[j][i]` is 32-bit signed (Q14)
- Product is 64-bit signed (Q28)
- After `>>> 14`, you get Q14 result
- Accumulating many such values still needs 64-bit to avoid overflow

### ReLU

Trivial in hardware:
```verilog
assign relu_out = (signed_input > 0) ? signed_input : 32'd0;
// Or equivalently: check MSB (sign bit)
assign relu_out = signed_input[31] ? 32'd0 : signed_input;
```

### Argmax (Output Layer)

Compare 10 Q14 values, select the index of the largest. No division/softmax needed.

### Memory Requirements

| Layer | Weights (words) | Biases (words) | Input Buffer | Output Buffer |
|-------|-----------------|----------------|-------------|--------------|
| 0     | 100,352         | 128            | 784         | 128          |
| 1     | 8,192           | 64             | 128         | 64           |
| 2     | 2,048           | 32             | 64          | 32           |
| 3     | 320             | 10             | 32          | 10           |

**Total weight storage:** ~111K × 32-bit = ~444 KB

### Suggested Hardware Approaches

1. **Fully Parallel:** Instantiate all neurons simultaneously. Fastest (1-cycle per layer)
   but requires enormous area. Only feasible for smaller layers (2, 3).

2. **Time-Multiplexed:** One MAC unit per layer, iterate over neurons and inputs.
   Much smaller area, but slower (`input_size × output_size` cycles per layer).

3. **Hybrid:** Parallelize within a layer (e.g., 10-way parallel for output neurons)
   and serialize across inputs. Good balance of speed vs. area.

### Hardware Debugging (4-Layer)

During simulation with Cadence Xcelium (`xrun`), two critical bugs were diagnosed and fixed to achieve a perfect 100% bit-true match with the C Reference model:

1. **Dequantize Precision Drift**: During batch testing of all 10 digit classes, a subtle precision drift emerged (e.g., C max-score `205637` vs RTL `195954`), which caused a failed prediction on Class 8. We determined that the SystemVerilog `dequantize` logic used "round-towards-zero", whereas C's `>> 14` cast operates as a strict arithmetic right-shift (floor toward negative infinity). By simplifying `nn_pkg.sv` to cleanly apply `>>> 14` directly, the precision perfectly aligned across the 300+ neurons to the exact integer.
2. **FSM Array Shifting (Pixel Drop) Bug**: A massive bug was hidden in the autogenerated `nn_top.sv` FSM. The initial Two-Process FSM contained a redundant `S_FIFO_PREFETCH` state which mistakenly popped `fifo_buf[0]` from the FIFO one cycle early, shifting the entire 784-pixel array by 1 index (horizontally shifting and wrapping the 2D image!). Astonishingly, the NN was robust enough to still achieve 90% test accuracy on the skewed input vector.
   - **The Fix**: Spliced out `S_FIFO_PREFETCH` and jumped straight from `S_IDLE` to `S_START_L0`. Then corrected the FIFO read bounds in `S_RUN_L0` from `l0_cnt < LAYER0_IN-2` to `LAYER0_IN-1`. This resulted in exactly 784 read triggers, perfectly emptying the FIFO, clearing phantom second-inferences from UVM, and achieving absolute **100% test passing accuracy** exactly mirroring the C model down to the last decimal.

## Architecture Comparison: Our Design vs. Wei-In's SystemVerilog NPU

Comparing our 4-Layer NN architecture to the [SystemVerilog-Portfolio-main NPU](https://github.com/weiinlai/SystemVerilog-Portfolio/tree/main/Synthesis):

### 1. Topology & Capacity
- **Wei-In's NPU**: `784 -> 30 -> 30 -> 10 -> 10`. Uses very narrow hidden layers (30 neurons), resulting in a smaller area but limiting the model's capacity to learn complex features.
- **Our NN**: `784 -> 128 -> 64 -> 32 -> 10`. Uses much wider layers, providing higher model capacity and finer feature extraction at the cost of a larger area footprint.

### 2. Activation Function (Hardware Cost)
- **Wei-In's NPU**: Uses **Sigmoid**. Requires a Look-Up Table (LUT) ROM (`sig_rom.sv`) to approximate the exponential function, which consumes block RAM/area.
- **Our NN**: Uses **ReLU**. Highly modernized and hardware-efficient. Implemented with a simple zero-clamping multiplexer (`(in > 0) ? in : 0`), saving significant area and eliminating ROM latency.

### 3. Fixed-Point Precision & Accumulation
- **Wei-In's NPU**: Uses **Q1.15** (16-bit) precision with saturating logic to prevent additions from overflowing.
- **Our NN**: Uses **Q14** mapped to **32-bit** datapaths, and expands to **64-bit** (`[2*DATA_WIDTH-1:0]`) during MAC accumulation before right-shifting. This provides a massive dynamic range that naturally avoids overflow without complex saturation hardware.

### 4. RTL Modularity & SystemVerilog Features
- **Wei-In's NPU**: Leverages advanced SV features like `Interfaces` and `Structs` to clean up routing. However, the layers are hardcoded as separate files (`Layer1.sv`, `Layer2.sv`, etc.).
- **Our NN**: Emphasizes **Highly Parameterized code generation**. We use a single `layer.sv` module combined with `generate-for` loops and a Python automation script (`gen_weights_and_rtl.py`). This script dynamically generates the 234 neuron instantiations and splits the memory files, achieving ultimate scalability (e.g., trivially expanding to an 8-layer network configuration with zero RTL manual rewrites).
